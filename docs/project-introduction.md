# 无人信息领域智能情报分析平台技术方案

---

## 1. 项目背景与需求分析

### 1.1 行业背景

近年来，无人机、无人车、无人船、无人潜航器等各类无人装备在全球范围内快速发展。各主要国家及头部企业持续加大投入，技术路线呈现多元化格局，集中式控制架构与分布式架构各有侧重，自主导航、集群智能、通信组网、动力能源等方向不断产出新成果。无人系统已从单一平台的遥控操作阶段，逐步进入多平台协同和自主决策的新阶段。随之而来的是海量的技术文献、试验数据和评估报告，给研究分析工作带来了不小的压力。

这个领域有几个突出的特点。信息量大、更新快、来源杂。一份技术评估报告动辄几百页，一个方向可能有十几家机构同时在研究，每个月都有新的白皮书、技术通报、论文和会议报告发布。一个中等规模的研究团队，一年积累的各类资料能达到上千份。资料格式也不统一，Word、PDF、Excel、扫描图片都有，分散在不同的系统和目录里，彼此之间没有关联，缺少统一管理。无人系统的技术迭代周期还在缩短，数字孪生、边缘计算等新概念不断涌现，分析人员必须持续跟踪前沿动态，研究成果才能跟上节奏。

从国际竞争格局看，无人系统已是科技竞争的重要方向。各国在无人机蜂群作战、水下无人潜航器编队、空地协同无人集群等方向投入大量资源。能否及时准确地掌握各方技术进展、研判发展趋势、识别潜在风险，直接影响研究工作的价值和时效性。

### 1.2 现状问题分析

当前无人系统研究人员面临的主要矛盾，已经从"找不到资料"变成了"资料太多，消化不过来"。具体来说：

多源信息交叉验证困难。分析人员通常要同时跟踪十几个技术方向，每周阅读几十份报告。不同来源的报告因为评估方法、数据来源和分析视角不同，对同一个问题经常给出矛盾的结论。比如集群智能的实用化时间表，不同机构的预测差异明显；自主导航技术的成熟度评估，不同测试标准下的结论也大相径庭。靠人工逐份比对，效率低，还容易漏。资料越多，矛盾被遗漏的概率就越大。

资料分散，检索效率低。研究资料散落在邮件附件、个人终端、历史文件夹、内部网盘等各处，没有统一索引。想找一份特定资料，经常要在好几个系统里来回翻。如果是跨领域、跨时段的信息整合，比如"近三年水下无人装备通信组网技术的发展脉络"，人工检索的难度更大。

研究成果难以复用。由于没有统一的知识管理机制，前期分析成果很难被后来的人用上。同类课题重复做资料收集和分析的情况普遍存在。比如，某位分析人员三年前做过的自主导航技术路线分析质量很高，但因为没有可检索的成果归档，接手类似课题的同事根本不知道，只能从头来过。

报告质量参差不齐。不同分析人员的知识储备和分析方法有差异，同一个课题出来的报告深度和质量可能差别很大。没有标准化的分析流程和工具支撑，报告质量主要靠个人能力，团队层面很难做到质量均一。

### 1.3 建设需求

传统的资料管理工具，包括文档库、网盘、知识库，解决的是"存"和"找"的问题，解决不了"深度理解"和"辅助研判"的问题。分析人员需要一套智能化的专业平台，具备以下能力：

多格式资料统一管理。支持 Word、PDF、Excel、图片、OFD 等多种格式的上传和管理，不需要人工预处理。系统能自动识别文档结构，提取关键信息，把非结构化资料转成结构化的知识条目。

基于专业资料库的智能问答。能基于团队积累的资料回答专业问题，答案可以追溯到具体资料和章节。支持多轮对话和上下文关联，从简单的事实查询到复杂的对比分析都能处理。

自动矛盾检测。能自动识别多份资料之间的结论矛盾和数据不一致，对矛盾进行分级分类，帮助管理人员按优先级处置。

智能报告生成。能综合多份资料的观点，自动生成结构化研究报告初稿，覆盖技术对比、趋势研判、风险识别和态势分析等研究类型。

研究成果持续积累。各阶段的分析成果，包括问答记录、分析报告、知识词条等，都能沉淀下来供后续检索和复用，形成团队知识的正向积累。

### 1.4 建设必要性

当前市面上的通用大语言模型无法满足上述需求，原因有以下几点。

领域知识局限。通用大模型的知识基于训练数据，有时间截断，不知道用户团队的内部报告和最新行业动态，只能给出泛化的回答。无人系统是一个高度专业化且快速迭代的领域，通用模型的知识储备远远不够。

输出可靠性不足。大语言模型存在"幻觉"问题，会生成看起来专业但缺乏事实依据的内容，结论无法溯源和验证。在研究决策场景中，这类不可靠的输出可能导致错误判断。如果涉及国防科技等敏感领域，后果更严重。

缺乏多文档交叉分析能力。通用大模型没法同时读取和分析用户积累的大量专业资料，做不了多源信息的矛盾检测和交叉验证。上下文窗口有限，处理不了几十份报告的综合分析。

资料管理缺失。通用大模型不具备资料管理功能。研究资料散落在各处，资料之间的关系完全靠人脑记忆。

研究成果无法沉淀。通用大模型的对话记录没有结构化管理，分析成果留不下来，后来的人用不上。

数据安全风险。通用大模型通常需要联网调用外部 API，用户输入的资料和问题会传到第三方服务器，敏感领域有数据泄露风险。

综上，本项目需要建设一套面向无人系统研究场景的专业化智能情报分析平台。无人系统领域技术迭代快、信息更新频繁的特点，决定了传统的人工工作模式已经跟不上节奏，必须借助智能化工具把分析人员从资料整理中解放出来，专注于需要专业判断的工作。

## 2. 技术架构概述

### 2.1 总体架构

平台采用分层架构，自下而上分为数据层、服务层和应用层，各层职责明确、松耦合，方便扩展和维护。

数据层负责资料的存储与管理，包括关系型数据库（结构化业务数据）、向量数据库（文本语义索引和相似度检索）、文件存储系统（原始文档持久化保存）。

服务层是平台的核心计算引擎，包括文档解析服务、智能问答服务、矛盾检测服务、报告生成服务和知识图谱服务。各服务模块化独立部署，通过标准内部接口通信，可以单独升级和扩展。

应用层面向用户，提供前台分析工作空间和后台管理工作空间的 Web 界面。前端采用响应式设计，适配主流浏览器。

### 2.2 核心技术：Graph-RAG

平台的智能分析能力基于 Graph-RAG（图增强检索增强生成）技术构建。传统 RAG 方案在回答问题时，只是从文档中检索出零散的文本片段交给大模型，模型看到的是一个个孤立的"碎片"，理解不了资料之间的关联。Graph-RAG 在此基础上引入了知识图谱，把资料中的关键概念（技术方向、装备型号、研究机构、项目名称等）和它们之间的关系（引用、分类、对比等）组织成一张网络。提问时，系统不仅检索相关文本，还能沿着知识图谱找到关联信息，回答更全面、更有深度。

这项技术让平台有了三项关键能力：一是能自动发现多份资料之间的结论矛盾，因为不同资料的观点在图谱中彼此关联，系统可以直接比对；二是能支撑跨资料的综合分析，因为图谱把分散在不同文档中的信息串联了起来；三是能随着资料积累不断完善知识网络，越用越强。

### 2.3 数据安全架构

平台在架构层面贯彻数据安全优先原则。所有数据存储和计算都在用户本地完成，不依赖外部网络。前后端通过标准化 API 接口通信，支持基于角色的访问控制（RBAC），不同权限的用户只能访问职责范围内的数据和功能。敏感数据在传输和存储过程中均加密处理。

## 3. 平台总体方案

### 3.1 平台定位

无人信息领域智能情报分析平台是面向无人系统研究分析人员的专业化工作平台，核心功能包括四个方面：

资料管理。统一管理所有无人系统相关的研究报告、动态信息、翻译文献和图表数据，提供高效检索。平台不只是资料仓库，而是经过解析、分类和结构化处理的知识资产管理系统。

智能问答。基于专业资料库回答领域问题，所有结论可追溯到具体资料来源。问答能力覆盖从简单事实查询到复杂多文档综合分析。

矛盾检测。自动识别多份资料之间的结论矛盾和数据不一致，标注并呈现差异分析。这是平台区别于传统资料管理工具和通用 AI 问答工具的核心能力。

报告生成。综合多份资料观点，自动生成结构化研究报告，覆盖技术对比、趋势研判、风险识别和态势分析四类研究场景。

### 3.2 双空间架构设计

平台采用前台与后台双空间架构。

前台是分析工作空间，面向一线分析人员，提供资料检索、智能问答、综合研究等功能。前台展示的所有内容都经过后台审核，信息质量有保障。分析人员不用关心数据管理的复杂性，在一个干净、可信的环境中工作。

后台是管理工作空间，面向管理人员，提供资料上传与审核、项目信息维护、组织架构管理、系统配置等功能。后台是平台数据质量的管控中枢，管理人员在这里完成资料入库、审核发布、冲突处置等日常工作。

两个空间共享统一的数据底层，实时同步。后台改了数据，前台马上能看到；前台的问答记录，后台也能查到。前后台形成"供料—用料"的循环：后台负责上传、审核和发布，前台负责查阅、分析和应用，双方协同提升数据质量。

典型应用场景：管理人员在后台上传一份技术通报并审核通过后，分析人员在前台马上就能看到，也能用智能问答查里面的内容。同时管理人员可以看到分析人员的高频问题，发现哪个方向的资料问得多但库里还缺，就能有针对性地补充。这种闭环设计让平台数据始终贴合业务需求，不会出现"建了不用、用了不准"的情况。

### 3.3 数据安全保障

平台支持完全本地化部署，所有数据存储和分析计算都在用户自有服务器上完成，不涉及外部数据传输。

完全本地化部署。平台不依赖外部云服务，断网环境下也能正常运行。所有资料存在用户自己的服务器上，数据不出网络边界。对涉及国防科技等敏感领域的研究工作，这一点至关重要。

前后台权限隔离。前台分析人员只能看到审核通过的资料，接触不到后台管理功能和待审核内容。管理人员通过后台掌控数据发布权限。权限体系支持细粒度控制，可按岗位灵活配置。

数据存储可靠性。平台采用数据库事务机制保障数据一致性，意外断电或系统异常时不会丢数据。同时支持定期备份，确保极端情况下的数据可恢复。

### 3.4 与传统 RAG 方案的技术对比

传统 RAG（检索增强生成）方案有明显的技术局限，本平台做了针对性升级：

| 对比维度 | 传统 RAG 方案 | 本平台方案 |
|---------|-------------|----------|
| 文件格式支持 | 仅支持纯文本及简单 PDF | 支持 Word、PDF、TXT、OFD、图片（OCR）、Excel（保留结构）等 |
| 数据源接入 | 通常只对接单一数据源 | 支持本地文件系统、数据库、外部系统等 |
| 文档理解能力 | 基于文本片段切分的向量检索，缺乏结构化理解 | 智能识别文档结构，提取标题、作者、关键数据、表格和图表说明，生成结构化知识词条 |
| 关联分析能力 | 仅支持单文档内容查询 | 支持多文档交叉比对、矛盾检测、趋势分析、技术对比等 |
| 成果积累机制 | 无持久化管理，对话结束即丢失 | 结构化管理所有分析成果，支持检索复用 |
| 质量管控 | 无审核机制 | 前后台分离加审核发布流程，确保信息质量 |

## 4. 前台功能详述

前台首页提供四个功能入口，对应分析人员的四个核心工作场景：资料查阅、资料上传、智能问答、综合研究。四个模块覆盖了从"获取资料"到"产出报告"的完整链路。

### 4.1 研究报告

研究报告库是平台的资料检索中心，汇集了所有经过系统解析和审核入库的研究资料。对分析人员来说，这是获取权威信息的第一站，也是日常使用频率最高的模块之一。

#### 4.1.1 检索与筛选

分析人员可通过关键词搜索定位目标资料，系统支持多维度组合筛选：

领域分类方面，支持空中无人装备、地面无人装备、水面无人装备、水下无人装备等类别。报告类型方面，支持对比分析、趋势研判、风险评估等。关键词搜索支持"自主导航"、"集群智能"等技术方向关键词，系统基于语义理解匹配，不限于精确词汇匹配。时间范围方面，支持按入库时间或发布时间筛选。

多个筛选条件可叠加使用，搜索结果支持分页浏览和按相关性或时间排序。资料库积累到一定规模后，精准筛选能力是提升效率的关键。

#### 4.1.2 报告详情

点击报告标题进入详情页，展示完整信息：

基本信息包括标题、领域分类（一级标签和二级标签）、报告类型、入库时间、关键词标签。扩展信息包括作者、摘要、页数等补充字段，支持自定义扩展。文件路径指向原始文件存储位置，方便追溯查阅。

详情页有查看和编辑两种模式。查看模式信息只读，适合快速浏览；编辑模式可修正分类、关键词等，修改即时生效。分析人员发现分类不准或关键词需要补充时，可以直接在前台改，不用切到后台。

#### 4.1.3 资料覆盖范围

研究报告库不只是放狭义的"报告"，而是涵盖多种资料类型：

研究报告是关于自主导航、集群智能、通信组网、动力能源等技术的正式研究成果，通常篇幅长、分析深入，是资料库的核心。动态信息是领域最新消息、技术突破、项目进展等时效性内容，篇幅短但更新快。译丛译著是国外研究资料的翻译版本，保留原文和译文信息，方便对照查阅。图表数据包括技术对比图、发展路线图、性能指标图表等可视化素材。

所有资料入库前都要经过统一的解析和分类流程。前台只展示已完成解析和审核的资料，处理中或待审核的资料不会出现，从机制上杜绝未经把关的信息干扰分析判断。

#### 4.1.4 典型应用场景

场景一：技术方向调研。分析人员接到调研任务，用关键词搜索几秒内获取资料列表，按领域和报告类型筛选，逐份查阅，快速完成资料梳理。以前需要翻好几个文件夹和系统才能凑齐的资料，现在一个页面就能全部获取。

场景二：报告撰写资料准备。比如要写"集群智能技术现状"报告，用"集群智能"关键词检索出所有相关资料，再筛出"对比分析"和"风险评估"两类，几分钟就能梳理完资料清单。传统方式可能要花半天以上。

场景三：跨领域信息整合。写涉及多个方向的综合性报告时，按自主导航、集群智能、通信组网、动力能源等方向分别检索，快速汇总。统一分类体系保证了跨领域检索的一致性。

### 4.2 资料上传与智能提取

该功能让分析人员把各类格式资料上传到平台，系统自动提取关键信息并转为结构化知识词条。

#### 4.2.1 多格式支持

平台支持 Word、PDF、TXT、OFD、图片、Excel 等多种格式的上传和解析。不同格式的解析引擎有针对性优化：Word 保留段落结构和样式，PDF 支持文字层提取和 OCR 双层处理，图片通过 OCR 识别文字，Excel 保留表格结构和数据关系。

#### 4.2.2 图片/表格上传与 OCR 识别

平台支持图片和表格的专项上传功能，具备 OCR（光学字符识别）能力。上传表格图片时，系统自动调用 SiliconFlow OCR API 识别图片中的表格结构，将图片内容转换为 Markdown 格式的表格数据。转换后的 Markdown 表格可直接用于后续的知识管理和智能问答。

支持的图片格式包括 PNG、JPG、JPEG 等常见格式。OCR 处理采用异步任务队列，上传后立即返回任务 ID，前端轮询任务状态直到处理完成。

#### 4.2.3 多张表格图片合并

针对同一份表格被拆分为多张图片的场景（如长表格分页拍摄），平台提供多张表格图片合并功能。用户可选择多张 OCR 识别后的表格图片，系统将各张图片识别出的 Markdown 表格数据合并为一份完整的表格。合并过程会自动处理表头重复、行列对齐等问题。

#### 4.2.4 文档自动命名

上传文档时，系统自动生成统一格式的文档名称，命名规则为：

```
{项目名}_{库名}_{序号}
```

例如：`无人集群_研究报告_001`、`通信组网_图表库_003`。该命名规则确保了文档命名的一致性和可追溯性，避免命名混乱。批量上传时同样遵循此命名规则，系统自动递增序号。

#### 4.2.5 智能信息提取

上传后系统自动提取以下信息：文档元数据（标题、作者、创建时间、格式等）；技术领域识别（自动匹配到平台的分类体系）；关键结论和数据（技术成熟度预测、性能参数、时间节点等）；引用关系（识别文档引用的其他资料）。

提取的信息经结构化整理后生成知识词条，包含标题、摘要、分类和引用来源，方便后续检索和引用。

#### 4.2.6 自定义字段

上传时可添加自定义扩展字段，如来源机构、发布时间、密级、项目编号等。自定义字段机制让平台能适应不同机构的业务规范，不改系统底层就能扩展信息维度。

#### 4.2.7 审核流程

上传资料经系统解析后以"待审核"状态进入后台，管理人员审核内容、确认分类、补充信息后才能发布到前台。系统提取的信息是"初稿"，人工审核是质量把关的第一道防线。

效率提升示例：一份百余页的 PDF 格式国外技术评估报告，传统方式需要人工花几天提取内容和分类标注。用平台的智能提取功能，系统自动完成信息提取和词条生成，管理人员几分钟审核确认，整个过程从几天缩短到半小时以内。按年均处理上千份资料的团队算，仅入库环节就能省出不少人力。

### 4.3 智能问答

智能问答是平台使用频率最高的功能，采用基于 Graph-RAG 的检索机制，所有回答都基于真实资料，结论可溯源、可验证，避免了通用大模型的"幻觉"问题。

#### 4.3.1 问答机制

系统收到提问后，依次执行三步：

语义检索。基于语义理解从资料库中检索相关内容，支持跨词汇匹配。比如用户问"无人机自主飞行能力发展到什么阶段"，系统能匹配到使用"自主导航技术成熟度"、"飞行自主性评估"等不同表述的资料。

交叉验证与综合分析。多份资料涉及同一主题时，系统自动比对各方说法，找出一致结论和分歧点。这让回答不只"有出处"，而且"经过验证"。

生成回答并标注来源。回答中明确标注引用的资料和具体章节，用户可点击追溯。每条回答附带置信度评分，评分越高说明依据越充分、观点越一致，分析人员可据此判断答案的可靠程度。

#### 4.3.2 富文本渲染与图片显示

智能问答的回答内容支持完整的 Markdown 渲染，包括标题层级、列表、代码块、表格、加粗/斜体等格式。当回答中引用了资料库中的图片（如技术对比图、架构图等），系统会在回答中直接内嵌显示相关图片，而非仅给出图片链接。图片显示支持点击放大查看。

#### 4.3.3 引用来源面板

参考 CitedReferencesPanel 设计，智能问答页面右侧提供引用来源面板。当系统生成回答后，所有被引用的资料来源以结构化卡片形式展示在面板中，每张卡片包含文档标题、相关段落摘要、相似度评分等信息。用户点击任意引用卡片可快速跳转到原始资料详情页，实现答案的溯源验证。引用来源面板让问答结果的可信度透明可见。

#### 4.3.4 问答参数配置

管理人员可在系统设置中调整智能问答的检索参数：

图片/表格相似度阈值。控制检索结果中图片和表格的匹配精度，阈值越高返回的结果越精准但数量越少，阈值越低返回的结果越多但相关性可能下降。

Top-K 配置。控制每次问答返回的引用来源数量上限，默认值为 5。可根据资料库规模和分析需求灵活调整。

上述参数调整后即时生效，无需重启服务。

#### 4.3.5 典型应用场景

"自主导航技术当前发展水平如何？" 系统综合多份报告给出概览，覆盖技术成熟度、主要方案、最新进展等。"固定翼无人机与旋翼无人机的优劣势对比？" 系统从对比分析报告中提取结论，结构化呈现各方案适用场景。"水下无人装备当前主要技术瓶颈？" 系统归纳风险识别报告中的发现，按严重程度排列。"各国在集群智能方面的技术路线差异？" 系统综合多份报告做横向对比，呈现不同国家的技术选择和依据。"某份评估报告的核心结论是什么？" 系统直接定位到目标报告，提取核心观点。

#### 4.3.6 会话管理

系统支持多轮连续对话，每个新话题可建独立对话，不同话题的历史互不干扰。会话记录持久化保存，关闭页面或退出登录都不丢失。

多轮对话支持渐进式深入：分析人员可以从宽泛的问题入手，根据回答逐步追问细节，形成由面到点的分析路径。这比单次搜索更贴合实际研究中的思维过程。

#### 4.3.7 快捷提问

页面底部预设高频问题入口，点击即可获取答案。预设问题可根据使用情况定期更新，保持与当前业务关注点一致。

### 4.4 智能综合分析与报告生成

该模块面向复杂研究任务，能综合多份资料观点自动生成结构化研究报告。智能问答适合"问一个问题、拿一个答案"的场景，本模块适合"多维度、系统性分析"的深度研究。实际工作中，分析人员通常先通过智能问答拿到具体答案，需要对某一方向做全面分析时再切换到本模块。

#### 4.4.1 四类研究场景

技术对比分析，是对不同技术路线或方案做横向比较。系统从多份资料中提取关键指标，分析技术成熟度、优劣势和适用场景，以对比表格或分析段落呈现。适用于多条技术路线并存时的决策支持。比如比较"视觉-惯性融合导航"和"地形匹配导航"两种自主导航路线，或比较"集中式控制"和"分布式控制"两种集群智能架构。

发展趋势研判，是基于多份资料做纵向分析，梳理特定技术的发展脉络、关键拐点和未来走向。系统比对不同时间点的资料论述变化，识别加速趋势、放缓趋势和新兴变量。适用于中长期规划和技术路线预判。比如分析"集群智能"从实验室验证到工程化应用的轨迹，或评估"氢燃料电池"在无人装备中的应用前景。

风险点识别，是自动检测多份资料之间的结论矛盾和数据不一致。系统标注矛盾点，分析差异成因（假设条件不同、测试环境不同、评估方法不同等），给出处置建议。每条冲突记录标注严重程度等级（严重/高度/中度）。比如 A 报告预测某型无人机自主导航技术 2027 年成熟，B 报告说至少要到 2030 年，系统自动识别该矛盾并分析原因。

综合态势分析，是对无人系统宏观方向做全局态势扫描，综合技术发展、装备布局、项目进展、国际竞争态势等多维度信息，形成整体性判断。适用于年度综述、战略评估等需要全局视野的研究任务。

#### 4.4.2 报告生成流程

分析人员选定研究场景和分析对象后，系统从资料库检索所有相关资料，执行交叉比对和综合分析，生成结构化报告初稿。初稿通常包含研究背景与范围界定、核心发现与分析、多源资料交叉验证结果、矛盾与不确定性说明、结论与建议。

分析人员对初稿做审核，验证准确性、补充遗漏、评估结论，可根据专业判断修改和补充。审核通过后发布到前台。这种"系统出初稿、人工审核"的协作模式，系统负责大规模资料处理，最终判断权在分析人员手里。

#### 4.4.3 报告质量保障

引用溯源方面，每份报告逐一标注引用资料来源（文档名称、章节位置），支持点击跳转到原始资料核实。状态管理方面，报告列表展示类型和完成状态，支持按类型和状态筛选。审核机制方面，管理人员逐份审核报告质量，只有审核通过的才发布到前台。

### 4.5 深度研究（DeepResearch）

深度研究功能是面向复杂研究任务的进阶分析工具，区别于智能问答的单次问答模式和综合分析的结构化报告模式。深度研究支持对某一课题进行多轮、渐进式的深入探索，系统会自动拆解研究问题、分步检索资料、综合多轮发现，最终形成系统性的研究结论。

深度研究的核心特点包括：

多步推理。系统不是一次性检索和回答，而是将复杂问题分解为多个子问题，逐步深入探索。每个子问题的检索和分析结果都会作为下一步的输入，形成递进式的分析链路。

跨资料综合。深度研究过程中，系统会同时检索和阅读多份相关资料，自动比对不同来源的观点和数据，识别共识和分歧。

过程可视化。研究的每一步推理过程对用户可见，用户可以了解系统是如何从问题出发、经过哪些检索和推理步骤、最终得出结论的，增强了研究过程的透明度和可信度。

深度研究适用于需要全面、系统分析的复杂课题，如"某技术领域未来五年的发展趋势及影响因素分析"、"多国在某技术方向的技术路线差异及成因"等。

## 5. 后台功能详述

后台是平台的数据管理中心，负责资料管理、组织维护和系统配置。左侧导航菜单有 9 个功能模块：信息库（下设 4 个子库）、项目库、组织架构库、智能问答管理、综合研究管理、结论冲突、方向建议、知识图谱管理和系统配置。

后台首页有统计面板，展示资料总数、项目总数、组织机构数量等指标。面板还提供近期数据变化趋势，如本周新增资料数、待审核资料数、活跃冲突数等，帮管理人员快速识别需要处理的事项。

### 5.1 信息库

信息库是平台的数据基础，按资料类型分为四个子库。四个子库共享统一的管理框架和操作规范，管理人员可在同一界面内完成各类资料管理。

#### 5.1.1 研究报告库

以表格形式展示所有研究报告，支持关键词搜索、领域分类筛选和状态筛选（处理中/待审核/已解析），支持分页浏览。

这是管理人员审核资料质量的主要工作场所。新上传的资料解析完后以"处理中"或"待审核"状态进入本库，管理人员审核内容、确认分类、补充信息后改为"已解析"，资料才在前台展示。日常工作流程是：先处理待审核的新资料，逐份审核；再检查已有资料的分类是否需要调整。

#### 5.1.2 动态信息库

收录时效性较强的信息，如新型装备发布、关键技术突破、国际合作项目启动、重要会议召开等。管理人员可通过弹窗快速新增动态条目，也可通过上传页面提交带附件的完整资料。

动态信息库的价值在于快。管理人员即时录入最新动态，分析人员当天就能在前台看到，比研究报告的审核周期短得多。

#### 5.1.3 译丛译著库

以卡片形式展示翻译文献，呈现标题、作者、译者、领域分类和内容简介。详情页支持完整信息编辑，包括原文标题、原文语种、出版机构、翻译完成度等专有字段。

在无人系统领域，大量研究资料以英文或其他语言发表。译丛译著库让翻译成果有系统化管理，避免了不同人员对同一资料重复翻译的浪费。

#### 5.1.4 图表库

以画廊缩略图形式展示可视化素材，包括技术对比图、发展路线图、系统架构图、性能指标图表等。支持放大查看和详情编辑。图表来源信息（摘自哪份报告、由谁上传）完整保留。

#### 5.1.5 统一详情页设计

四个子库用统一的详情页模板和操作规范。编辑模式支持修改标题、分类、状态、关键词和自定义扩展字段，可按资料类型灵活配置补充信息（研究报告的"页数"、译著的"原文语种"、图表的"分辨率"等）。管理人员在不同子库间切换时不需要重新适应。

### 5.2 项目库

项目库以卡片形式管理所有研究项目，每张卡片展示项目名称、描述、所属组织、负责人、技术领域、优先级和当前状态。顶部工具栏支持按组织、领域、状态和关键词组合筛选。

#### 5.2.1 项目全生命周期管理

支持从立项到结题的全过程管理。新建时指定基本信息和归属组织，中途可调整负责人、优先级和归属，结题后更新状态归档。删除操作需二次确认。项目的全过程记录为后续复盘和经验总结提供了信息基础。

#### 5.2.2 典型应用场景

项目盘点：按状态筛选"进行中"项目，一览年度立项数量、方向分布和进展状况。负荷分析：按组织筛选，评估各研究所项目承载量，识别资源分配不均。责任追溯：按项目名称检索，获取负责人和归属组织信息。

#### 5.2.3 与组织架构库的双向关联

项目库和组织架构库数据互通。项目卡片标注归属组织，组织架构库中各部门展示关联项目。管理人员可以从"人"的角度看项目分布，也可以从"事"的角度追溯责任。

### 5.3 组织架构库

组织架构库以树形结构展示研究机构的完整组织体系，支持部门、研究所、实验室三种类型。每个节点展示名称、描述、人员编制、关联项目数和负责人信息。

新增组织时选择类型并指定上级，系统自动挂到对应位置。编辑可改名称、描述和负责人。删除有前置校验：下级有子组织时不允许删除，须先清理。每个组织节点有"查看项目"入口，跳转到项目库看该组织的全部项目。

右侧统计面板展示负责人列表（标注所属部门）和组织总览（总人数、研究所数、实验室数、项目总数）。

### 5.4 智能问答管理

该模块让管理人员查看所有人的问答历史。每条记录包含提问者、问题内容、系统回答、时间和置信度评分。

管理人员主要用在三个场景：抽查回答质量，发现问题反馈给技术团队优化；回溯分析过程，查阅历史问答了解前期思路和结论；识别需求热点，分析高频问题分布，发现哪个方向问得多但资料少，就优先补充。

### 5.5 综合研究管理

统一管理四类智能分析报告，支持按报告类型和完成状态筛选。管理人员可查看详细内容、审核质量并追踪进度。报告从"进行中"到"已完成"的全过程清晰可见，方便管理层总体把控。

### 5.6 结论冲突

结论冲突检测在资料入库后自动执行交叉比对，识别多份资料之间的矛盾和数据不一致。每条冲突记录包含：涉及矛盾的资料清单和各方结论（冲突来源）；矛盾类型和具体差异量（差异分析）；颜色标识的严重程度分级，红色严重、橙色高度、蓝色中度（严重程度分级）；系统给出的采信建议（处置建议）。

管理人员逐条审查，决定采信方案或安排补充研究。这个功能把矛盾发现从传统的最终报告撰写阶段前移到资料入库阶段。

传统方式下，矛盾检测完全靠分析人员的记忆和笔记。每周读几十份资料，不可能每份都记得清清楚楚，遗漏矛盾的概率很高。平台的自动检测不会遗忘和遗漏，所有资料入库后即时完成比对。资料库越大，潜在的交叉矛盾越多，人工检测的难度指数级上升，而平台的检测能力可以线性扩展。

### 5.7 方向建议

方向建议模块基于资料库数据分析，为研究规划提供决策支持。系统分析各方向的研究覆盖度、空白领域和待解决矛盾，自动生成方向建议。每条建议包含三部分：现状分析（该方向已做的工作和尚未覆盖的领域）；可选方向（多个可深入的方向及其研究价值）；可行性评估（各方向的实施难度和所需条件）。

比如资料库中"自主导航"方向已经积累了技术现状分析、方案对比、风险识别报告等，而"通信组网"方向只有少量翻译文章，缺乏系统性分析。系统会建议优先安排通信组网的技术梳理。所有建议基于资料库实际数据生成，不是凭经验判断。

### 5.8 知识图谱管理

知识图谱以可视化网络图展示资料之间的关联关系。前台为分析人员提供知识点关系网络视图，呈现技术、装备、机构、项目之间的关联结构。分析人员可以快速识别核心概念、关键机构和它们之间的关系。

后台管理功能包括：全局知识网络视图，支持全屏查看、缩放和拖拽；社区检测，自动识别天然聚类的知识点群组；图谱洞察，自动识别潜在结构问题，如孤立节点（尚未建立关联的方向）、稀疏连接区域（知识板块间缺少桥梁）、核心枢纽节点（高关联度，需重点关注）。

这些洞察帮助管理人员把握知识体系结构，找出"该连没连"的薄弱环节，为后续资料补充和知识整理提供方向。

### 5.9 系统配置

系统配置模块用于调整平台运行参数。知识图谱权重系数支持对不同类型关联关系设置差异化权重，如"直接引用"权重高于"分类相同"，调整后前台展示即时更新。平台基本信息支持修改名称、版本号等。

#### 5.9.1 问答参数配置

智能问答模块的检索参数可在系统设置中调整：

图片相似度阈值。控制问答回答中引用图片的匹配精度。阈值范围 0.0~1.0，默认值 0.7。阈值越高，返回的图片越精准但数量越少；阈值越低，返回的图片越多但相关性可能下降。

表格相似度阈值。控制问答回答中引用表格数据的匹配精度。配置方式与图片相似度阈值一致。

Top-K 配置。控制每次问答返回的引用来源数量上限，默认值为 5。增大该值可提供更多参考来源，但会增加响应时间。

#### 5.9.2 OCR 配置

OCR 处理相关参数支持在系统设置中调整，包括 OCR API 服务端点、超时时间等配置项。

## 6. 项目价值总结

无人信息领域智能情报分析平台要解决的核心问题是：让分析人员从资料整理中解放出来，专注于需要专业判断的工作。平台不是替代分析人员的能力，而是让个人能力和系统能力形成更好的组合。

### 6.1 工作流程优化

传统工作流程是这样的：拿到研究课题后，多渠道搜集资料，逐份阅读，手工做笔记，人工比对多源结论，自行汇总分析，最后撰写报告。全流程耗时数天到数周，还有遗漏重要信息和矛盾的风险。

平台支撑下的工作流程变成：在研究报告库检索关键词，用智能问答获取精准答案，让系统生成报告初稿，然后人工审核和补充。全流程从数天缩短到数小时，而且系统全覆盖、无遗漏的特点保障了分析质量。分析人员可以把更多时间投入到深度分析和专业判断上。

### 6.2 核心能力矩阵

| 业务痛点 | 平台解决方案 | 预期效果 |
|---------|------------|---------|
| 资料量大、检索困难 | 智能语义检索，多维度组合筛选 | 资料定位从小时级缩短至秒级 |
| 多源信息结论矛盾 | 自动矛盾检测，分级分类管理 | 矛盾发现从报告撰写阶段前移至入库阶段 |
| 同类课题重复研究 | 研究成果持续积累，支持检索复用 | 消除重复劳动 |
| 综合报告撰写耗时 | 多资料综合分析，自动生成初稿 | 报告初稿从数天缩短至数分钟 |
| 资料格式多样 | 多格式上传，智能解析提取 | 入库处理从数天缩短至半小时以内 |
| 数据安全要求高 | 完全本地化部署，数据不出网络边界 | 满足涉密环境合规要求 |
| 报告质量参差不齐 | 标准化分析流程加引用溯源机制 | 提升报告质量一致性和可追溯性 |

### 6.3 长期价值

长期来看，平台的价值不只是提升单次研究效率，更在于团队知识资产的持续积累。随着资料库不断增长、知识图谱不断完善、分析成果不断沉淀，平台会成为团队的知识基础设施。每一份入库资料、每一次有价值的问答、每一份审核过的报告，都在为这个知识体系增值。这种积累是传统分散式工作模式做不到的。
